## 1. 项目概述

### 1.1 项目目标

构建一个**浏览器兴趣分析器（InterestProfiler）**，能够：

1. 自动识别并读取用户浏览器历史记录（PC端）
2. **通过平台网页爬虫获取完整的跨设备观看历史**（移动端 + PC端 + Pad端 + TV端）
3. 智能筛选出所有视频网站访问记录
4. 基于 AI/NLP 技术对观看内容进行多维度分类
5. 生成可视化的兴趣分析报告

### 1.2 核心价值

- **隐私优先**：所有数据本地处理，不上传云端；Cookie 仅用于请求平台自身 API
- **多浏览器支持**：覆盖 Chrome / Edge / Firefox / Brave
- **跨设备覆盖**：浏览器历史（PC）+ 平台网页爬虫（移动端/多设备）= 完整视图
- **智能分类**：结合标题 NLP、URL 特征、平台元数据（分区标签、UP主信息）多维度推断
- **可视化输出**：交互式 HTML 报告 + 导出能力

### 1.3 为什么需要平台网页爬虫？

| 数据来源 | 覆盖范围 | 局限性 |
|----------|----------|--------|
| **浏览器本地历史** | 当前PC浏览器的记录 | 仅PC端；URL可能不完整；无视频元数据 |
| **B站历史API** | **全部设备**的历史：手机端(`dt=1,3,5,7`)、Pad端(`dt=4,6`)、TV端(`dt=33`)、Web端(`dt=2`) | 需登录Cookie；API频率限制 |
| **抖音历史API** | 全部设备的浏览记录 | 需登录Cookie；反爬签名机制 |

> 💡 浏览器历史中 B站/抖音只占用户在这些平台总观看量的 **一小部分** — 大部分观看发生在手机 App。爬平台历史页面可以获得用户在所有设备上的完整观看轨迹。

### 1.3 适用场景

| 场景 | 描述 |
|------|------|
| 个人兴趣发现 | 用户了解自己的视频消费图谱 |
| 家长管控分析 | 了解孩子在上网时观看的内容类型 |
| 团队文化分析 | 分析团队成员的共同兴趣倾向 |
| 用户研究 | 为产品设计提供用户行为数据参考 |

## 2. 需求分析

### 2.1 功能需求（Functional Requirements）

#### FR1: 浏览器自动检测

- 自动扫描系统已安装的浏览器（Chrome / Edge / Firefox / Brave / Opera）
- 支持用户手动指定浏览器历史记录路径
- 识别每个浏览器的 Profile 目录（Default / Profile 1 / ...）

#### FR2: 历史记录提取

- 读取浏览器 SQLite 数据库（`History` / `places.sqlite`）
- 处理浏览器运行时文件锁：拷贝副本后读取
- 提取字段：`url`, `title`, `visit_count`, `last_visit_time`, `visit_duration`
- 时间戳转换：Chromium WebKit 时间 → Unix 时间戳 → 可读日期

#### FR3: 视频网站识别

- 内置主流视频平台域名规则库（30+ 站点）
- 支持自定义域名/URL 模式扩展
- 区分视频页面与非视频页面（如首页、消息中心等）

#### FR4: 视频内容分类

- 主分类（一级）：游戏、科技、教育、音乐、娱乐、生活、体育、新闻、影视、动漫、知识
- 子分类（二级）：如 游戏→FPS / MOBA / 单机 / 独立游戏
- 分类置信度标注（高 / 中 / 低）

#### FR5: 兴趣分析报告

- 视频平台分布饼图（区分设备来源）
- 内容类型分布柱状图
- 观看时长趋势折线图
- Top-N 频道/UP主排名
- 兴趣标签词云
- 设备来源分布（PC / Mobile / Pad / TV）
- HTML 交互式报告 + JSON 数据导出

#### FR6: 浏览器Cookie提取与登录检测

- 从浏览器 Cookie 数据库中提取目标平台的登录 Cookie
- 检测用户是否已登录 B站 / 抖音（通过 API 探针请求）
- 未登录时友好提示，引导用户登录后重试
- 支持 Chromium 系和 Firefox 的 Cookie 存储格式

#### FR7: B站历史记录网页爬取

- 调用 B站历史API `api.bilibili.com/x/web-interface/history/cursor`
- 支持 Cursor 分页全量拉取（直到 `list` 为空）
- 提取字段：标题、BV号、UP主、分区标签、观看时间、观看进度、设备类型
- 支持视频/直播/专栏三种内容类型的过滤

#### FR8: 抖音浏览记录网页爬取

- 调用抖音历史API `douyin.com/aweme/v1/web/history/read/`
- 支持 Cursor 分页全量拉取（直到 `has_more=false`）
- 处理反爬签名（X-Bogus / a-bogus 参数）
- 提取字段：视频标题、作者、观看时间

### 2.2 非功能需求（Non-Functional Requirements）

| 需求项 | 指标 |
|--------|------|
| **性能** | 10,000 条历史记录处理 < 3 秒（不含 AI 分类）；API 爬取单平台 < 30 秒 |
| **隐私** | 所有数据本地处理，不上传原始 URL/标题；Cookie 仅用于目标平台 API 调用 |
| **兼容性** | Windows 10+ / macOS 12+ / Linux（Ubuntu 20.04+） |
| **可扩展性** | 视频站点规则、分类体系、爬虫平台支持热加载 |
| **准确性** | AI 分类准确率 > 80%（基于标题+NLP） |
| **容错性** | 爬虫失败不影响浏览器历史分析；任一数据源失败自动降级

## 3. 现有数据摸底

基于对实际 Edge 浏览器数据的分析（验证环境：Windows 11 + Edge 浏览器）：

### 3.1 数据库 Schema 验证

**`urls` 表**（核心数据源）：

```sql
CREATE TABLE urls (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    url             LONGVARCHAR,     -- 完整URL
    title           LONGVARCHAR,     -- 页面标题
    visit_count     INTEGER DEFAULT 0 NOT NULL,
    typed_count     INTEGER DEFAULT 0 NOT NULL,
    last_visit_time INTEGER NOT NULL, -- Chromium WebKit时间戳
    hidden          INTEGER DEFAULT 0 NOT NULL
);
```

**`visits` 表**（访问详情）：

```sql
CREATE TABLE visits (
    id              INTEGER,
    url             INTEGER,         -- 关联 urls.id
    visit_time      INTEGER,         -- 访问时间
    visit_duration  INTEGER,         -- 停留时长（微秒）
    transition      INTEGER,         -- 跳转类型
    -- ... 其他字段
);
```

**`keyword_search_terms` 表**（搜索关键词）：

```sql
CREATE TABLE keyword_search_terms (
    keyword_id      INTEGER,
    url_id          INTEGER,         -- 关联 urls.id
    term            LONGVARCHAR,     -- 原始搜索词
    normalized_term LONGVARCHAR      -- 标准化搜索词
);
```

## 4. 技术选型

### 4.1 总体技术栈

```

│                    技术栈总览 (v2.0)                  │
├───────────────┬─────────────────────────────────────┤
│ 语言          │ Python 3.11+                        │

│ 依赖管理      │ Poetry / pip + requirements.txt      │

```

### 4.2 核心依赖选型对比

#### 4.2.1 AI 分类方案对比

| 方案 | 技术 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **A. 本地向量模型** | sentence-transformers + 预定义类别向量 | 完全离线、速度快、零成本 | 分类粒度有限、需预定义类别 | ⭐ **推荐主力** |
| **B. LLM API** | OpenAI / Claude / 本地 LLM | 理解力强、分类细腻 | 成本高、有隐私风险、依赖网络 | 高价值精细化分析 |
| **C. 规则引擎** | 正则 + 关键词匹配 | 极快、无依赖 | 覆盖率低、维护成本高 | 快速预筛、兜底 |
| **D. 视频平台 API** | B站 History API / 抖音 History API | 精确标签、跨设备数据 | 频率限制、需登录Cookie | ⭐ **主要数据增强** |

**推荐策略：A (主) + C (兜底) + D (平台API增强) + B (可选增强)**

```
标题输入 → 规则引擎预筛(确定高置信度标签) 
         → 本地向量模型(计算与类别中心向量的余弦相似度)
         → [可选] LLM二次校验低置信度结果

数据输入 → 浏览器历史(PC) + B站History API(全设备) + 抖音History API(全设备)
         → 数据合并去重 → 分类管道 → 报告
```

#### 4.2.2 爬虫引擎选型对比

| 方案 | 技术 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|------|
| **A. 纯 API 调用** | requests + Cookie | 轻量、速度快 | 无法处理JS渲染、可能缺签名 | ⭐ **B站主力** |
| **B. 浏览器模拟** | Playwright / Selenium | 完整浏览器环境、自动处理签名 | 重量级、慢、资源占用高 | 兜底方案 |
| **C. 混合方案** | curl_cffi (TLS指纹) + requests | 绕过TLS指纹检测、接近API速度 | 仍需手动处理签名 | ⭐ **抖音主力** |

**推荐策略：B站用 A（API直调）；抖音用 C（curl_cffi + 签名工具）；Playwright 作为最后兜底**

```python
class BiliScraper:
    """B站历史记录爬虫"""
    
    API_URL = "https://api.bilibili.com/x/web-interface/history/cursor"
    BATCH_SIZE = 30  # 每页最大30条
    
    def __init__(self, sessdata: str):
        self.session = requests.Session()
        self.session.headers.update({
            "User-Agent": "Mozilla/5.0 ... Chrome/130.0.0.0 ...",
            "Referer": "https://www.bilibili.com/",
        })
        self.session.cookies.set("SESSDATA", sessdata, domain=".bilibili.com")
    
    def fetch_all(self, business_types: List[str] = None) -> pd.DataFrame:
        """
        全量拉取历史记录
        
        对于每种business类型(archive/pgc/live/article)分别拉取,
        直到 list 为空或长度 < BATCH_SIZE
        """
        all_records = []
        for biz in (business_types or ["archive", "pgc", "live", "article"]):
            cursor = {"max": 0, "view_at": 0, "business": ""}
            while True:
                params = {
                    "max": str(cursor["max"]),
                    "view_at": str(cursor["view_at"]),
                    "business": biz,
                    "ps": self.BATCH_SIZE,
                    "type": ""  # 空=全部
                }
                resp = self.session.get(self.API_URL, params=params, timeout=15)
                data = resp.json()
                
                if data["code"] != 0:
                    if data["code"] == -101:
                        raise NotLoggedInError("B站未登录")
                    break
                
                items = data["data"]["list"]
                if not items:
                    break  # 无更多数据
                
                for item in items:
                    all_records.append({
                        "title": item["title"],
                        "author": item["author_name"],
                        "author_id": item["author_mid"],
                        "tag": item.get("tag_name", ""),  # B站自带分类！
                        "bvid": item["history"].get("bvid", ""),
                        "view_at": item["view_at"],
                        "progress": item.get("progress", 0),
                        "duration": item.get("duration", 0),
                        "device": item["history"].get("dt", 0),  # 设备类型
                        "business": item["history"]["business"],
                        "platform": "bilibili",
                        "source": "api",  # 标记来源
                    })
                
                if len(items) < self.BATCH_SIZE:
                    break  # 最后一页
                    
                cursor = data["data"]["cursor"]
                time.sleep(0.5)  # 尊重API，避免429
        
        return pd.DataFrame(all_records)
```

```python
class DouyinScraper:
    """抖音浏览记录爬虫"""
    
    API_URL = "https://www.douyin.com/aweme/v1/web/history/read/"
    
    def __init__(self, cookies: dict):
        """
        cookies 需要包含:
        - sessionid
        - passport_csrf_token  
        - ttwid (建议)
        """
        self.cookies = cookies
        self.session = self._build_session()
    
    def _build_session(self):
        """使用 curl_cffi 构建带TLS指纹的会话（绕过反爬）"""
        from curl_cffi import requests as cffi_requests
        session = cffi_requests.Session()
        # 模拟 Chrome 130 的 TLS 指纹
        session.impersonate = "chrome130"
        session.headers.update({
            "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...",
            "Referer": "https://www.douyin.com/user/self?showTab=record",
        })
        for key, value in self.cookies.items():
            session.cookies.set(key, value, domain=".douyin.com")
        return session
    
    def fetch_all(self) -> pd.DataFrame:
        """
        全量拉取浏览记录，处理 a-bogus 签名
        """
        all_records = []
        cursor = 0
        has_more = True
        
        while has_more:
            params = {
                "count": 20,
                "cursor": str(cursor),
            }
            
            # 生成签名
            signed_params = self._sign_params(params)
            
            try:
                resp = self.session.get(
                    self.API_URL, 
                    params=signed_params,
                    timeout=15
                )
                data = resp.json()
                
                if data.get("status_code") != 0:
                    if "登录" in str(data.get("status_msg", "")):
                        raise NotLoggedInError("抖音未登录")
                    raise APIError(f"抖音API错误: {data.get('status_msg')}")
                
                items = data.get("aweme_list", [])
                for item in items:
                    all_records.append({
                        "title": item.get("desc", ""),
                        "author": item.get("author", {}).get("nickname", ""),
                        "author_id": item.get("author", {}).get("uid", ""),
                        "aweme_id": item.get("aweme_id", ""),
                        "view_at": item.get("create_time", 0),
                        "duration": item.get("duration", 0) // 1000,  # ms → s
                        "platform": "douyin",
                        "source": "api",
                    })
                
                has_more = data.get("has_more", False)
                cursor = data.get("cursor", 0)
                
                if not items:
                    break
                    
                time.sleep(1.0)  # 抖音反爬严格，间隔长一些
                
            except Exception as e:
                logger.warning(f"抖音爬取出错: {e}, cursor={cursor}")
                break
        
        return pd.DataFrame(all_records)
    
    def _sign_params(self, params: dict) -> dict:
        """为请求参数添加 a-bogus 签名"""
        from urllib.parse import urlencode
        base_url = self.API_URL
        query_string = urlencode(params)
        full_url = f"{base_url}?{query_string}"
        
        try:
            a_bogus = DouyinSigner.generate_a_bogus(full_url, params)
            params["a_bogus"] = a_bogus
        except Exception:
            pass  # 签名失败则尝试无签名请求
        
        return params
```

            except Exception as e:
                self.errors[platform] = str(e)

## 8. Cookie 提取与登录检测（NEW）

### 8.1 为什么需要提取浏览器Cookie

平台历史API需要**登录态Cookie**才能访问。与其让用户手动从浏览器开发者工具复制Cookie，不如直接从浏览器Cookie数据库中自动提取——无感、安全、自动化。

### 8.2 Chromium Cookie 存储

**路径**: `%LOCALAPPDATA%\<browser>\User Data\<Profile>\Network\Cookies` (SQLite)

**数据库结构**:
```sql
CREATE TABLE cookies (
    host_key TEXT,           -- 域名 (如 .bilibili.com)
    name TEXT,               -- Cookie名称
    value BLOB,              -- ⚠️ 加密存储!
    encrypted_value BLOB,    -- 加密值
    path TEXT,
    expires_utc INTEGER,
    is_secure INTEGER,
    is_httponly INTEGER,
    ...
);
```

#### Cookie 加密与解密

从 Chromium v33+ 开始，Cookie 值使用 **AES-256-GCM** 加密：

```
加密密钥存储位置:
  Windows: %LOCALAPPDATA%\Google\Chrome\User Data\Local State (encrypted_key)
         → DPAPI解密 → AES密钥
  
  macOS:   Keychain → "Chrome Safe Storage"
  
  Linux:   GNOME Keyring / KWallet 或 "--password-store=basic"
```

```python
class CookieExtractor:
    """从浏览器提取并解密Cookie"""
    
    def extract(self, browser_path: str, domain: str) -> Dict[str, str]:
        """
        从指定浏览器提取特定域名的Cookie
        
        Args:
            browser_path: 浏览器Profile根路径
            domain: 目标域名 (如 ".bilibili.com")
        
        Returns:
            { "SESSDATA": "abc123", "bili_jct": "def456" }
        """
        # Step 1: 读取加密密钥
        key = self._get_encryption_key(browser_path)
        
        # Step 2: 读取Cookie数据库
        cookie_db = os.path.join(browser_path, "Network", "Cookies")
        if not os.path.exists(cookie_db):
            cookie_db = os.path.join(browser_path, "Cookies")  # 旧版本
        
        # Step 3: 拷贝副本（加锁处理）
        tmp_db = self._copy_db(cookie_db)
        
        # Step 4: 查询并解密
        conn = sqlite3.connect(tmp_db)
        cursor = conn.execute(
            "SELECT name, encrypted_value FROM cookies WHERE host_key LIKE ?",
            (f"%{domain}%",)
        )
        
        cookies = {}
        for name, encrypted_value in cursor.fetchall():
            decrypted = self._decrypt(key, encrypted_value)
            if decrypted:
                cookies[name] = decrypted
        
        conn.close()
        os.remove(tmp_db)
        return cookies
    
    def _get_encryption_key(self, browser_path: str) -> bytes:
        """从 Local State 提取并解密AES密钥 (Windows: DPAPI)"""
        with open(os.path.join(browser_path, "Local State")) as f:
            local_state = json.load(f)
        
        encrypted_key = base64.b64decode(
            local_state["os_crypt"]["encrypted_key"]
        )
        # Windows: DPAPI解密
        # 前5字节是 "DPAPI" 前缀
        encrypted_key = encrypted_key[5:]
        return CryptUnprotectData(encrypted_key, None, None, None, 0)[1]
    
    def _decrypt(self, key: bytes, encrypted_value: bytes) -> Optional[str]:
        """AES-256-GCM 解密单个Cookie值"""
        # v10 格式: 前3字节版本号 "v10", 然后12字节 nonce, 最后是密文+tag
        if encrypted_value[:3] == b"v10":
            nonce = encrypted_value[3:15]
            ciphertext = encrypted_value[15:]
            
            cipher = AES.new(key, AES.MODE_GCM, nonce=nonce)
            plaintext = cipher.decrypt_and_verify(ciphertext[:-16], ciphertext[-16:])
            return plaintext.decode("utf-8")
        
        # 旧版: Windows DPAPI encrypted
        return CryptUnprotectData(encrypted_value, None, None, None, 0)[1].decode("utf-8")
```

## 10. AI 分类引擎设计

### 8.1 推荐方案：本地向量模型作为主力

#### 模型选型

| 模型 | 维度 | 大小 | 中文支持 | 速度 |
|------|------|------|----------|------|
| `paraphrase-multilingual-MiniLM-L12-v2` | 384 | 118MB | ⭐⭐⭐⭐⭐ | 极快 |
| `distiluse-base-multilingual-cased-v2` | 512 | 482MB | ⭐⭐⭐⭐ | 快 |
| `BAAI/bge-small-zh-v1.5` | 512 | 96MB | ⭐⭐⭐⭐⭐ | 极快 |

**推荐**: `paraphrase-multilingual-MiniLM-L12-v2` — 尺寸小、速度快、中文效果好

#### 分类流程

```
Step 1: 文本预处理
  标题清洗 → 去除特殊字符 → 中文分词(jieba) → 关键词提取(TF-IDF)
  
Step 2: 规则引擎初筛
  关键词命中 → 高置信度标签 [跳过后续步骤]
  例: "Python教程" → 科技/编程 (置信度: 高)
  例: "【LOL】王者局" → 游戏/MOBA (置信度: 高)
  
Step 3: 向量相似度分类
  encode(标题) → 与11个一级类别锚点向量计算余弦相似度
  → Top-1 + Top-3 类别 + 相似度分数
  → 相似度 > 0.7 → 高置信度, 0.4-0.7 → 中置信度, <0.4 → 低置信度
  
Step 4: 二级分类（向量模型）
  encode(标题) → 与所属一级类别下的子类锚点向量计算相似度
  → 最高分子类别

Step 5 (可选): LLM复核
  低置信度样本 → 批量发送LLM → 获取精确分类 + 解释
  Prompt: "请将以下B站视频标题分类到: 游戏/科技/教育/...，输出JSON"
```

## 12. 数据合并与去重策略（NEW）

### 12.1 为什么需要合并去重

浏览器历史和平台API爬取的数据存在大量重叠。例如：
- 在PC上用Chrome看了B站视频 → 同时出现在浏览器历史和B站API中
- 在手机上用App看了同一视频 → 仅出现在B站API中
- 抖音视频的网页链接 → 出现在浏览器历史中，但更完整的元数据在API中

**目标**: 两条数据源取**并集**，以API数据为准做元数据增强，去重后输出干净数据集。

### 12.2 去重策略

```python
class DataMerger:
    """多源视频数据合并去重"""
    
    # 去重键优先级（由精确到模糊）
    DEDUP_KEYS = [
        "video_id",           # BV号 / aweme_id（最精确）
        "url_normalized",     # 标准化URL
        "title_author_hash",  # 标题+作者组合哈希（模糊匹配）
    ]
    
    def merge(self, browser_df: pd.DataFrame, api_dfs: Dict[str, pd.DataFrame]) -> pd.DataFrame:
        """
        合并浏览器历史和平台API数据
        
        规则:
        1. 以 api 数据优先级更高（元数据更完整）
        2. 相同 video_id 的记录合并，api 数据覆盖 browser 数据
        3. 对于 URL 匹配的记录，用 api 的 tag/author/duration 增强 browser 记录
        4. browser 独有的记录保留
        """
        all_records = []
        
        # Step 1: 以API数据为基础
        for platform, api_df in api_dfs.items():
            if api_df is not None and not api_df.empty:
                all_records.append(api_df)
        
        # Step 2: 浏览器历史中去重
        api_ids = set()
        for df in api_dfs.values():
            if df is not None and 'video_id' in df.columns:
                api_ids.update(df['video_id'].dropna().unique())
        
        browser_only = browser_df[
            ~browser_df['video_id'].isin(api_ids)
            if 'video_id' in browser_df.columns
            else pd.Series([True] * len(browser_df))
        ]
        
        if not browser_only.empty:
            all_records.append(browser_only)
        
        # Step 3: 拼接 + URL级别二次去重
        merged = pd.concat(all_records, ignore_index=True)
        
        # 标准化URL去重 (去掉tracking参数)
        merged['url_key'] = merged['url'].apply(self._normalize_url)
        merged = merged.drop_duplicates(subset=['url_key'], keep='first')
        
        return merged
    
    @staticmethod
    def _normalize_url(url: str) -> str:
        """URL标准化：去除追踪参数、统一HTTPS、去除尾部斜杠"""
        from urllib.parse import urlparse, urlunparse
        parsed = urlparse(url)
        # 去除常见的追踪参数
        tracking_params = {
            'spm_id_from', 'vd_source', 'from_tab_name', 
            'showTab', 'share_id', 'utm_source', 'utm_medium',
            'ref', 'source', 'track_id'
        }
        clean_query = '&'.join(
            f'{k}={v}' for k, v in parse_qs(parsed.query).items()
            if k not in tracking_params
        )
        clean = parsed._replace(query=clean_query, fragment='', scheme='https')
        return urlunparse(clean).rstrip('/')
```

### 12.3 数据优先级与冲突解决

| 字段 | 优先级 | 说明 |
|------|--------|------|
| `title` | API > Browser | API返回的标题更完整（含副标题） |
| `author` | API > Browser | 浏览器历史通常没有作者信息 |
| `tag` | API only | B站API的 `tag_name` 是官方分区标签 |
| `duration` | API > Browser | API返回原始时长，浏览器记录不准确 |
| `device` | API > Browser | API的 `dt` 字段精确标识设备 |
| `view_at` | API > Browser | API时间戳更精确 |
| `visit_count` | Browser only | 仅浏览器有访问次数统计 |

## 14. 风险评估与对策

| 风险 | 等级 | 影响 | 对策 |
|------|------|------|------|
| **浏览器锁定数据库** | 🔴 高 | 无法读取正在运行的浏览器历史 | `shutil.copy2` 到临时目录再读；提示用户关闭浏览器 |
| **平台Cookie过期/失效** | 🔴 高 | API爬取失败，无法获取全设备数据 | 登录检测前置；提示用户重新登录；爬虫失败不影响浏览器历史分析 |
| **抖音反爬升级** | 🔴 高 | 签名算法失效导致爬取失败 | 使用`curl_cffi`模拟TLS指纹；Playwright兜底；关注F2/douyin-api项目更新 |
| **隐私泄露** | 🔴 高 | 用户Cookie或浏览数据被上传到第三方 | 全程本地处理；Cookie仅用于目标平台API；LLM仅发送标题不含URL |
| **API频率限制(429)** | 🟡 中 | 爬取中断、数据不完整 | 请求间隔(0.5-1s)；断点续传；指数退避重试 |
| **Cookie解密失败** | 🟡 中 | 无法从浏览器提取登录Cookie | 多密钥方案尝试；Firefox直接读取备选；提示用户手动粘贴Cookie |
| **视频分类不准确** | 🟡 中 | 兴趣画像偏差 | 平台`tag_name`增强分类（B站官方分区）；三级分类管道；置信度标注 |
| **浏览器版本兼容** | 🟡 中 | DB schema / Cookie加密变化 | `PRAGMA table_info`动态适配；多解密方案回退 |
| **大量历史记录性能** | 🟡 中 | 分类速度慢 | 批量向量编码(batch_size=64)；缓存模型；采样模式 |
| **平台网页结构变更** | 🟡 中 | API端点或参数变化 | 持续关注bilibili-API-collect等社区项目更新 |
| **本地模型下载失败** | 🟢 低 | 向量分类不可用 | 回退到规则引擎+TF-IDF；提供模型手动下载说明 |

## 附录 B：Chromium 时间戳转换工具函数

```python
import datetime

def chromium_time_to_datetime(webkit_microseconds: int) -> datetime.datetime:
    """
    将 Chromium WebKit 时间戳转换为 Python datetime
    
    Chromium 时间戳: 自 1601-01-01 00:00:00 UTC 起的微秒数
    """
    if webkit_microseconds <= 0:
        return None
    # 1601-01-01 到 1970-01-01 的微秒差: 11644473600000000
    unix_microseconds = webkit_microseconds - 11644473600000000
    return datetime.datetime.fromtimestamp(unix_microseconds / 1_000_000)

def datetime_to_chromium_time(dt: datetime.datetime) -> int:
    """反向转换"""
    unix_microseconds = int(dt.timestamp() * 1_000_000)
    return unix_microseconds + 11644473600000000
```
